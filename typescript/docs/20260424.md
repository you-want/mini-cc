# 终端 UI 与性能优化版本

本次迭代主要围绕**终端用户交互体验（React/Ink UI）**、**核心启动性能优化**以及**单文件二进制打包（Bun 编译）**三个方向进行了深度的开发与重构。以下是本次更新的详细记录：

## 🚀 1. 新增功能 (Features)

### 1.1 终端 React 交互界面 (基于 Ink)
*   **虚拟滚动列表组件** (`VirtualMessageList.tsx`)：针对大模型动辄数千字的长上下文输出，自研了只渲染可视区域附近消息的虚拟列表组件，彻底解决了终端 UI 会因为渲染大量 Flexbox 节点而导致 CPU 飙升和卡死的性能问题。
*   **流式对话界面** (`App.tsx`)：集成了 `ink-text-input` 实现命令行内的对话输入，并且无缝对接底层的 `Agent`，实现了大模型打字机效果的实时呈现。
*   **进度条组件** (`ProgressBar.tsx`)：提供终端进度渲染功能，包含完成率和文本描述提示，可用于后续下载依赖或安装过程的可视化。

### 1.2 极速冷启动架构 (Fast Startup Architecture)
*   **启动性能探针分析器** (`startupProfiler.ts`)：设计了轻量级的性能探针机制。应用在不同生命周期打点，通过 `--profile` 参数启动时可直接输出可视化的启动各阶段耗时表格，精准定位性能瓶颈。
*   **极速通道 (Fast-path)** (`cli.ts`)：针对 `--version` 和 `--health` 等简单命令，在最顶层直接拦截并退出。跳过了所有庞大依赖树（如 React、SDK）的解析，实现了“秒开”。
*   **并行预加载 (Parallel Pre-fetching)**：在主线程进行大量文件 I/O（`import` 解析）的同时，将全局配置加载等网络或磁盘 I/O 任务抛入 libuv 线程池后台并行处理，极致压缩冷启动耗时。
*   **防吞键机制 (Early Input Capture)**：在进程初始阶段抢先接管 `stdin` 缓存用户的按键输入，等到 React UI 完全加载并渲染完毕后再将按键重放，防止启动过程中的输入丢失。

### 1.3 遥测、隐私与特性开关
*   **隐私控制层** (`privacyLevel.ts`)：新增基于 `MINI_CC_TELEMETRY` 环境变量控制的隐私分级枚举（`ALLOW_TELEMETRY` 和 `STRICT_LOCAL`）。
*   **第一方事件记录** (`firstPartyEventLogger.ts`)：封装了模拟 OpenTelemetry 的事件打点上报接口，包含针对 `app_start` 和 `app_exit` 等核心动作的埋点。
*   **动态特性开关** (`getDynamicConfig`)：设计了获取 GrowthBook 等远程特性的接口，方便进行灰度发布或 A/B 测试实验。

### 1.4 本地测试与打包体系
*   **本地开发测试入口** (`demo-ui.tsx`)：新增了直接在本地调试 React UI 的沙箱命令 `pnpm run demo:ui`。
*   **Jest UI 自动化测试**：引入了 `ink-testing-library`，为虚拟滚动和进度条组件编写了可靠的单元测试并已全部通过。
*   **Bun 独立二进制打包** (`package.json`)：增加了 `pnpm run build:bun` 脚本，可将整个项目代码及其庞杂的 `node_modules` 编译为一个单独的、无外部依赖的二进制文件。

## 🛠️ 2. 修复与调整 (Fixes & Modifications)

### 2.1 依赖与环境兼容性修复
*   **修复 Bun 打包 CJS 模块兼容性问题**：
    *   **布局引擎崩溃**：在使用 Bun 编译时，底层的 `yoga-layout-prebuilt`（Ink 依赖）在严格模式下因未声明变量而抛出 `ReferenceError: _a is not defined`。已在 `cli.ts` 顶层注入 `global._a = undefined` 作为兼容补丁。
    *   **React 组件导出错误**：由于打包工具对 CommonJS 模块处理逻辑差异，`ink-text-input` 被包装进了 `default` 属性中，导致 React 抛出 `Element type is invalid`。已在 `App.tsx` 中编写动态导入类型判定，兼容了两种加载模式。
*   **屏蔽 Node 弃用警告**：在 `cli.ts` 顶层拦截了 Node.js v21+ 发出的 `punycode` 弃用警告，防止其污染原本整洁的终端命令行界面。

### 2.2 逻辑与代码结构优化
*   **全量中文注释**：为最近引入的架构层代码（`startupProfiler.ts`、`cli.ts`、`main.ts` 等）补充了极为详尽的中文开发注释，极大降低了项目代码的理解门槛。
*   **去除导致终端卡死的布局属性**：修改了 `VirtualMessageList` 中的占位符（`topSpacer`）并去除了外层会引起终端计算死循环的 `flexGrow` 属性，彻底解决了长时间运行或快速刷新时的“假死”问题。
*   **清理了未使用的变量**：修复了 `agent.getHistory` 中废弃参数引起的 Linter 检查报错。
