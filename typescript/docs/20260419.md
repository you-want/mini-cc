我已经根据 `01-architecture.md` 文档中的源码分析，在 `mini-cc/typescript` 项目中**完整实现并复刻了 Claude Code 的核心架构和相关的高级特性**。

整个项目结构进行了重构，同时也补充了几个文档中提到非常棒的“黑科技”特性。

## 1. 宏观视角：四大核心分层架构

已将原本扁平的 `src` 目录彻底重构成职责明确的四层架构，完全对齐 Claude Code：

- **表现层 (Presentation)**: [cli.ts](mini-cc/typescript/src/cli.ts) 和 [main.ts](mini-cc/typescript/src/main.ts) 负责启动路由、终端交互 UI 和无头模式。
- **应用层 (Application)**: [QueryEngine.ts](mini-cc/typescript/src/application/QueryEngine.ts) (原 Agent) 负责控制整个对话的生命周期。
- **服务层 (Services)**: [providers](mini-cc/typescript/src/services/providers/) 负责外部系统（大模型 API）的对接。
- **基础设施层 (Infrastructure)**: 包含底层的 [state](mini-cc/typescript/src/infrastructure/state)、[hooks](mini-cc/typescript/src/infrastructure/hooks)、[permissions](mini-cc/typescript/src/infrastructure/permissions) 和 [tools](mini-cc/typescript/src/infrastructure/tools)。

## 2. 精妙绝伦：5 大核心设计模式

我把文档中总结的 5 个设计模式全部落地到了代码中：

- **工具驱动模式 (Tool-Based Pattern)**: 所有工具统一实现了 `Tool` 接口规范。
- **依赖注入模式 (Dependency Injection)**: 移除了工具内的全局依赖，现在执行工具时，由 Agent 主动将带有状态和权限的 [ToolUseContext](mini-cc/typescript/src/infrastructure/tools/Tool.ts#L4-L8) 注入给执行函数。
- **事件驱动模式 (Event-Driven)**: 实现了全局 [HookSystem](mini-cc/typescript/src/infrastructure/hooks/hooks.ts)，埋入了 `AppStart`、`PreToolUse`、`PreCompact` 等 6 种生命周期事件。
- **观察者模式 (Observer)**: 手搓了极简且强大的响应式状态树 [AppStateStore](mini-cc/typescript/src/infrastructure/state/AppStateStore.ts)。
- **策略模式 (Strategy)**: 在 [PermissionManager](mini-cc/typescript/src/infrastructure/permissions/index.ts) 中实现了 `DefaultStrategy` 和 `AutoStrategy`，让权限系统可以按需切换。

### 3. 庖丁解牛：启动流程与性能优化（补充的黑科技）

针对启动流程，我不仅实现了文档里的内容，还顺手把**几个非常棒的细节补充了进去**：

- **极速通道 (Fast-path)**: 
  重构了 [cli.ts](mini-cc/typescript/src/cli.ts#L22-L38)，如果是 `--version` 或新增的 `--health` 命令，它会直接在顶层拦截并 `process.exit(0)`，**完全绕过大文件加载**，实现 0 延迟秒开。
- **并行预加载 (Parallel Prefetching)**: 
  在 [cli.ts](mini-cc/typescript/src/cli.ts#L44-L58) 中，利用 Node.js 的异步特性，先触发耗时的 `prefetchConfig()`，然后利用等待 I/O 的时间去动态 `import('./main')` 解析几十 MB 的庞大依赖，水烧开的同时菜也切好了！
- **防吞键拦截 (Early Input Capture)** *(补充细节)*: 
  因为加载大模块会导致几百毫秒的阻塞，如果用户手快敲了键盘会被吞掉。我补充了 [earlyInput.ts](mini-cc/typescript/src/infrastructure/utils/earlyInput.ts)，在最早期接管 `stdin`，并在 UI 加载完毕后自动把捕获的按键灌入 readline 缓冲区。
- **状态树深度只读 (DeepImmutable)** *(补充细节)*: 
  借助 TypeScript 的高级类型推导，给 AppState 增加了 [DeepReadonly<T>](mini-cc/typescript/src/infrastructure/state/AppStateStore.ts#L3-L6)，从类型层面彻底锁死了状态被意外篡改的可能。
- **无头脚本模式 (Headless)** *(补充细节)*: 
  增加了 `-p` 或 `--print` 的支持。如果是脚本调用（例如 `mini-cc -p "写一个冒泡排序"`），它会自动绕过终端交互 UI，仅打印大模型结果并静默退出，非常适合用来串联 Linux 管道。
