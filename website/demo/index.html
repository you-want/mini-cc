核心特性

为什么选择 mini-cc？

极简架构，极致性能。mini-cc 以 Rust 的"所有权"与"无畏并发"为基石， 打造了一个轻量却强大的 AI 编程智能体。

内存安全 & 零开销

避免繁重的面向对象系统，状态安全由生命周期与借用检查器严格保证，无 GC 停顿，无数据竞争。

SSE 流式输出

通过 eventsource-stream 深度解析大模型的打字机流式响应，包括思维链 (Reasoning Content) 的实时展示。

Tool Use 自动循环

内置 BashTool、FileReadTool、FileWriteTool，模型可自主决策调用工具，多轮循环直至任务完成。

多模型兼容

支持 OpenAI 兼容接口，可无缝切换 Qwen、GPT-4、Claude 等多种大模型，灵活配置 Base URL。

纯函数式架构

延续"极简"基因，无冗余抽象层，核心逻辑清晰可追溯，易于二次开发与定制扩展。

原生二进制

编译为原生机器码，启动迅速，内存占用极低，无需虚拟机或解释器，部署即运行。

架构设计

核心 工作流

mini-cc 的核心是一个简洁而强大的 Agent 决策循环， 通过工具调用与大模型推理的紧密配合，实现自主编程。

1

用户输入

用户在终端输入自然语言指令，如"帮我创建一个 Web 项目"

2

LLM 推理

大模型分析意图，生成 Tool Use 调用或最终回复，支持思维链推理

3

工具执行

根据模型决策，调用 BashTool / FileReadTool / FileWriteTool 执行操作

4

结果反馈

将工具执行结果返回给模型，模型判断是否继续循环或输出最终答案

Rust src/core/agent.rs 
pub async fn agent_loop(prompt: &str) -> Result<String> {
    let mut messages = init_messages(prompt);
    
    loop {
        // 1. 调用 LLM 获取响应
        let response = client.chat_completion(&messages).await?;
        
        // 2. 解析 Tool Use 调用
        if let Some(tool_calls) = response.tool_calls {
            for call in tool_calls {
                let result = execute_tool(&call).await?;
                messages.push(result);
            }
            continue; // 继续循环
        }
        
        // 3. 无工具调用，返回最终结果
        return Ok(response.content);
    }
}
内置工具

三大核心 Tool

mini-cc 内置三个核心工具，覆盖编程智能体的基本操作需求。

BashTool

利用 std::process::Command 高效且安全地执行 Shell 命令，支持命令输出捕获与错误处理。

BashTool("cargo build --release")

FileReadTool

基于 std::fs 实现文件读取，超大文件自动行数截断，避免内存溢出。

FileReadTool("src/main.rs")

FileWriteTool

使用 std::path::Path 和 create_dir_all 安全地自动补全目录并写入文件内容。

FileWriteTool("src/lib.rs", content)
快速开始

三步 启动 mini-cc

只需几分钟，即可在你的机器上运行 mini-cc 智能体。

01

环境准备

确保系统已安装 Rust 1.70+（推荐使用 rustup 安装）

curl --proto '=https' --tlsv1.2 -sSf https://sh.rustup.rs | sh
02

配置环境变量

在 .env 文件中填入你的 API Key 和模型配置

OPENAI_API_KEY="sk-xxxxxx"
OPENAI_BASE_URL="https://dashscope.aliyuncs.com/compatible-mode/v1"
MODEL_NAME="qwen-max"
03

编译并运行

使用 Cargo 构建并启动 Agent

cargo run
